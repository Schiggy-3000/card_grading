# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate testing and deployment via GitHub Actions. On every push to `master`:
1. Run backend unit tests → deploy both Cloud Functions to GCP (if tests pass)
2. Build the Vite frontend → deploy to GitHub Pages

**GCP Project:** `card-grading-492013`  
**GCP Region:** `europe-west1` (Belgium — GCF 2nd gen free tier applies in all regions)  
**Service Account:** `service-account-card-grading@card-grading-492013.iam.gserviceaccount.com`

## Architecture Overview

```
push to master
  │
  ├── [Job: backend-deploy]
  │     1. Checkout
  │     2. Set up Python 3.9
  │     3. pip install pytest + backend dev deps
  │     4. pytest backend/recognize/tests/ backend/grade/tests/
  │     5. Authenticate to GCP (Workload Identity or SA key)
  │     6. gcloud functions deploy recognize  (backend/recognize/)
  │     7. gcloud functions deploy grade      (backend/grade/)
  │     8. Output deployed URLs → set as step outputs
  │
  └── [Job: frontend-deploy]
        needs: backend-deploy  (so VITE_* URLs are known)
        1. Checkout
        2. Set up Node 20
        3. npm ci (frontend/)
        4. npm run build with VITE_RECOGNIZE_URL + VITE_GRADE_URL
        5. Deploy frontend/dist/ to gh-pages branch via peaceiris/actions-gh-pages
```

**Why `frontend-deploy` depends on `backend-deploy`:** The Vite build bakes `VITE_RECOGNIZE_URL` and `VITE_GRADE_URL` into the static bundle at build time. The deployed GCF URLs are needed first. If both jobs run in parallel, the frontend would be built with stale or empty URLs.

**How to get the deployed GCF URLs** (after the first manual deploy or after `backend.yml` runs once):
```bash
gcloud functions describe recognize --region=europe-west1 --format="value(serviceConfig.uri)"
gcloud functions describe grade      --region=europe-west1 --format="value(serviceConfig.uri)"
```

**Note on `GCP_SERVICE_ACCOUNT_KEY` vs Workload Identity Federation:**  
Using a service account JSON key is the simplest approach for a personal project. WIF (keyless auth) is more secure for team environments. This plan uses SA key for simplicity.

## Task 1: Backend Workflow

**File:** `.github/workflows/backend.yml`

- [ ] **Step 1: Create `.github/workflows/` directory**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Create `backend.yml`**

```yaml
name: Backend — Test & Deploy

on:
  push:
    branches: [master]
    paths:
      - "backend/**"
      - ".github/workflows/backend.yml"

jobs:
  test-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Python 3.9
        uses: actions/setup-python@v5
        with:
          python-version: "3.9"

      - name: Install test dependencies
        run: |
          pip install pytest
          pip install -r backend/recognize/requirements.txt
          pip install -r backend/grade/requirements.txt

      - name: Run recognize unit tests
        run: pytest backend/recognize/tests/ -v
        env:
          PYTHONPATH: backend/recognize

      - name: Run grade unit tests
        run: pytest backend/grade/tests/ -v
        env:
          PYTHONPATH: backend/grade

      - name: Authenticate to Google Cloud
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SERVICE_ACCOUNT_KEY }}

      - name: Set up gcloud CLI
        uses: google-github-actions/setup-gcloud@v2

      - name: Deploy recognize function
        run: |
          gcloud functions deploy recognize \
            --gen2 \
            --runtime=python39 \
            --region=europe-west1 \
            --source=backend/recognize \
            --entry-point=recognize \
            --trigger-http \
            --allow-unauthenticated \
            --memory=512MB \
            --timeout=60s \
            --project=card-grading-492013

      - name: Deploy grade function
        run: |
          gcloud functions deploy grade \
            --gen2 \
            --runtime=python39 \
            --region=europe-west1 \
            --source=backend/grade \
            --entry-point=grade \
            --trigger-http \
            --allow-unauthenticated \
            --memory=512MB \
            --timeout=120s \
            --project=card-grading-492013
```

**Notes:**
- `--allow-unauthenticated` — required for the React frontend to call the functions from the browser without a Google identity token. This is intentional for a public read-only app.
- `--memory=512MB` for `grade` — OpenCV image processing needs more than the 256MB default.
- `--timeout=120s` for `grade` — OpenCV on a cold start can be slow; 120s is safe.
- `paths` filter — the workflow only runs when `backend/**` or itself changes. Frontend-only pushes skip this job.
- `PYTHONPATH` — pytest needs the function's own directory on the path (same as running locally from `backend/recognize/`).

## Task 2: Frontend Workflow

**File:** `.github/workflows/frontend.yml`

- [ ] **Step 1: Create `frontend.yml`**

```yaml
name: Frontend — Build & Deploy

on:
  push:
    branches: [master]
    paths:
      - "frontend/**"
      - ".github/workflows/frontend.yml"
      - ".github/workflows/backend.yml"  # re-deploy if backend URLs change

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Node 20
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        working-directory: frontend
        run: npm ci

      - name: Build
        working-directory: frontend
        run: npm run build
        env:
          VITE_RECOGNIZE_URL: ${{ secrets.VITE_RECOGNIZE_URL }}
          VITE_GRADE_URL: ${{ secrets.VITE_GRADE_URL }}

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: frontend/dist
          publish_branch: gh-pages
          force_orphan: true
```

**Notes:**
- `peaceiris/actions-gh-pages@v4` — pushes `frontend/dist/` to the `gh-pages` branch. GitHub Pages serves from that branch.
- `force_orphan: true` — the `gh-pages` branch has no meaningful git history; `force_orphan` keeps it as a single-commit branch, preventing the branch from growing unbounded.
- `GITHUB_TOKEN` is automatically provided by GitHub Actions — no secret to configure for Pages deployment.
- `cache-dependency-path: frontend/package-lock.json` — scopes the npm cache to the frontend subdirectory.
- Vite bakes `VITE_*` env vars into the bundle at build time (`import.meta.env.VITE_*`). They must be present during `npm run build`, not at runtime.
